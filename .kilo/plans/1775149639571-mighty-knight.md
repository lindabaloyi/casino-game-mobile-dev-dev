# Plan: Fix Home Page UI and Ensure Game Profile CRUD

## Problem Analysis
The home page UI is not displaying username and avatar from the database, and we need to ensure all CRUD functionality for game profiles (including username and avatar) is working properly.

### Current Issues Identified:
1. **Home Page Loading State**: The home screen checks `if (!profile)` but profile always exists with default values ("Player"), so loading state isn't handled properly
2. **Data Sync Issues**: Profile data may not be syncing correctly on app startup for authenticated users
3. **CRUD Verification**: Need to ensure username/avatar updates work end-to-end from client to server to database

## Solution Steps

### 1. Fix Home Screen Loading State
- Update `app/(tabs)/index.tsx` to properly handle loading states using `isLoading` from `usePlayerProfile`
- Show loading spinner while profile data is being fetched and synced
- Prevent displaying stale/default data during sync

### 2. Improve Profile Data Synchronization
- Ensure `usePlayerProfile` hook properly syncs authenticated user data on mount
- Fix any issues with the sync logic in `syncWithServer()` function
- Verify server response parsing for username and avatar fields

### 3. Verify Avatar Display Logic
- Confirm `currentAvatar` calculation in `useHomeScreen` correctly maps avatar IDs to emoji
- Ensure `AVATAR_OPTIONS` are properly exported and imported
- Test avatar display in both ProfileCard and HomeMenuModal components

### 4. Ensure CRUD Operations for Username/Avatar
- Verify client-side update functions (`updateUsername`, `updateAvatar`) work correctly
- Confirm server-side endpoints handle updates properly (PUT /api/profile)
- Test optimistic updates with rollback on failure
- Ensure changes sync bidirectionally between local storage and server

### 5. Test End-to-End Functionality
- Test home page displays correct username/avatar after login
- Verify profile updates persist across app restarts
- Test CRUD operations in profile editing screen
- Ensure guest users get proper default data

### 6. Add Error Handling and Fallbacks
- Add proper error states for failed profile loads
- Implement retry logic for sync failures
- Provide fallback display when data is unavailable

## Files to Modify
